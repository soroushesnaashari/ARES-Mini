ARES Mini is a lightweight, educational, and portfolio-friendly Retrieval-Augmented Generation (RAG) project inspired by the ideas behind the ARES paper. It is not a full reproduction of the paper. Instead, it is a simplified implementation that demonstrates the main pipeline of a modern RAG system and adds a basic evaluation layer to estimate answer quality.

The main purpose of ARES Mini is to show how a question-answering system can use external documents as knowledge sources instead of relying only on the language model’s internal memory. In this project, the user asks a question, the system searches a small local document collection stored in the data/docs folder, retrieves the most relevant document passage, sends that passage to a large language model, and then returns an answer based only on the retrieved context. After that, the project computes simple evaluation scores to show how relevant and faithful the answer is to the retrieved text.

ARES Mini is designed to be simple enough for a master’s portfolio project, but still rich enough to demonstrate important ideas from Retrieval-Augmented Generation. It shows the full flow from retrieval to generation to evaluation, which makes it a useful educational demo. The project can be run locally, tested through FastAPI’s interactive documentation page, and later deployed as a live demo on Hugging Face Spaces.

The project has three main components.

First, the retrieval component. This part reads documents from the local data/docs folder. In the current version, the project supports .txt, .pdf, and .json files. Each file is loaded and converted into text. Then the system uses TF-IDF vectorization and cosine similarity to compare the user’s question with the stored documents. The most relevant document is selected and returned as the context for answer generation. This means the system does not answer from random memory. It answers based on the most relevant external source it can find.

Second, the generation component. This part sends the question and retrieved context to an LLM API. In this project, the LLM is used only after retrieval. The model is instructed to answer using only the provided context and to say that it does not know if the context does not contain the answer. This is important because it makes the system more grounded and reduces hallucination. The language model is not used to search documents. Its job is only to write the final response in natural language.

Third, the evaluation component. This part scores the quality of the answer using a simple lightweight scoring method. It checks whether the answer is related to the question, whether it is faithful to the retrieved document, and whether the overall answer quality looks reasonable. This evaluation layer is what makes the project more than a normal RAG demo. It is inspired by the ARES paper, which focuses on automated evaluation of retrieval-augmented systems. However, ARES Mini uses a much simpler approach so that it remains easy to understand, easy to build, and suitable for a student project.

The project is intentionally small and practical. It does not train any custom judges, does not require a large dataset, and does not depend on heavy infrastructure. It is meant to be a clear demonstration of the core idea: retrieve relevant knowledge, generate an answer from that knowledge, and evaluate the result. This makes it useful both as a learning project and as a portfolio project.

The interface is built with FastAPI and includes a simple web frontend with a clean black-and-white design. The homepage allows a user to enter a question, see the retrieved document, read the generated answer, and inspect the evaluation scores. The goal is to keep the interface minimal while still making the workflow easy to understand.

In short, ARES Mini is a simplified RAG and evaluation system that demonstrates how external documents, a language model, and automatic scoring can work together in one pipeline. It is a practical, understandable, and visually clean project that shows the essential logic behind modern retrieval-augmented AI systems without making the implementation overly complicated.
